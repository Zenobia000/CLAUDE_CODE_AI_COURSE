## 🎭 視覺動線設計

```
1️⃣ 左上角 → 2️⃣ 右上角
    ↓           ↓
4️⃣ 左下角 ← 3️⃣ 右下角

```

### 漸進式資訊揭露
```
Phase 1: 吸引注意
- 大標題 + 視覺衝擊
- 核心價值主張
- 情境設定

Phase 2: 建立理解
- 概念解釋 + 視覺化

- 對比分析

Phase 3: 促進行動
- 實作步驟
- 立即可做的事情

```

```

- 關鍵框架解釋

- 實例分析
- 工具介紹
- 技術細節

低注意力時刻 (投影片後段):
- 總結回顧
```

## 🎯 互動設計原則

### 1. 認知負荷管理
```
簡化原則:
- 每張投影片最多 5-7 個要點
- 避免超過 3 個層級的資訊架構
- 使用視覺分組減少選擇壓力
- 一次只聚焦一個主要概念

漸進揭露:
- 複雜圖表分步驟顯示
- 代碼範例從簡單到複雜
- 概念從抽象到具體
- 從 Why → What → How 的順序
```

### 2. 視覺連貫性
```
模組間連接:
- 使用一致的轉場效果
- 模組顏色漸變過渡
- 保持核心設計元素
- 進度指示器連續性

頁面間關聯:
- 前後呼應的設計元素
- 一致的圖示語言
- 統一的文字層次
- 相似的佈局結構
```

### 3. 情感引導設計
```
開場設計 (建立期待):
- 震撼的視覺開場
- 清晰的價值承諾
- 學習目標的視覺化
- 時間投資回報預覽

過程設計 (維持參與):
- 定期的進度確認
- 小勝利的慶祝設計
- 挑戰的視覺化呈現
- 成就感的即時回饋

結尾設計 (促進行動):
- 成果總結的儀式感
- 下一步的明確指引
- 長期價值的願景描繪
- 立即行動的催化劑
```

## 🎬 動畫與轉場設計

```

時長: 2-3 秒
效果: 象徵工具選擇的機械精確

M02 → M03: 工具到記憶
動畫: 文件歸檔，色彩從紫轉青
時長: 2-3 秒
效果: 象徵資訊的有序存儲

M03 → M04: 記憶到安全
動畫: 護盾啟動，色彩從青轉紅
時長: 2-3 秒
效果: 象徵防護系統的建立

M04 → M05: 安全到行動
動畫: 綠燈亮起，色彩從紅轉綠
時長: 2-3 秒
效果: 象徵安全確認後的行動

M05 → M06: 行動到未來
動畫: 火箭升空，色彩從綠轉金
時長: 2-3 秒
效果: 象徵成長和價值實現
```

```
內容出現動畫:
- 淡入 (Fade In): 300ms, ease-out

- 縮放 (Scale): 400ms, elastic

重點強調動畫:
- 脈衝效果 (Pulse): 1s, infinite
- 晃動提示 (Shake): 500ms, attention
- 發光效果 (Glow): 2s, fade-in-out

進度動畫:
- 進度條填充: 1-2s, linear
- 勾選標記: 800ms, bounce
- 數字計數: 1.5s, ease-in-out
```

## 📊 品質控制檢核清單

### 視覺一致性檢查
```
色彩系統:
□ 每個模組使用正確主色
□ 輔助色彩保持一致
□ 文字對比度 ≥ 4.5:1
□ 色彩無障礙性符合 WCAG 標準

字體系統:
□ 標題字體一致 (Inter Bold)
□ 內文字體一致 (Inter Regular)
□ 代碼字體一致 (JetBrains Mono)
□ 字體大小階層正確

間距系統:
□ 8px 基準倍數間距
□ 一致的邊距設定
□ 適當的行高設定
□ 合理的元素間距
```

### 內容品質檢查
```
資訊架構:
□ 每張投影片單一重點
□ 邏輯流程清晰
□ 前後呼應完整
□ 學習目標一致

視覺效果:
□ 圖表清晰易懂
□ 動畫不過度干擾
□ 重點突出明顯
□ 整體和諧統一

技術品質:
□ 檔案格式標準化
□ 圖像解析度足夠
□ 字體嵌入正確
□ 相容性測試通過
```

### 使用者體驗檢查
```
可讀性:

□ 導航邏輯清晰

參與度:
□ 視覺吸引力足夠

□ 互動機會合理
□ 成就感設計到位

實用性:
□ 學習目標可達成

□ 價值主張突出
```
