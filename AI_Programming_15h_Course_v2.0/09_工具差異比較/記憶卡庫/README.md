# 記憶卡庫 - 工具選擇決策訓練

## 記憶卡設計理念

### 為什麼需要記憶卡？

```
傳統學習問題：
"知道"三大工具的差異 ≠ "會選"正確的工具

解決方案：
通過間隔重複（Spaced Repetition）建立快速決策能力
```

### 設計原則

```
❌ 不要：死記定義
Q: Claude Code 的 context window 是多大？
A: 200K tokens

✅ 要：情境決策
Q: 【情境】你需要重構一個 10 萬行的 Python 專案，有 2 週時間。選擇？
A: 【決策】Claude Code
   【理由】大規模需要 200K context 理解全局，EPCV 工作流提供系統性方法
   【記憶點】> 10 萬行 = 必須 Claude Code
```

---

## 記憶卡架構

### 分類體系

```
15 張記憶卡，分為 5 類：

類別 1: 規模判斷（3 張）
├─ 小專案（< 1K 行）
├─ 中專案（1K-10K 行）
└─ 大專案（> 10K 行）

類別 2: 任務類型（5 張）
├─ CRUD 開發
├─ 架構設計
├─ Bug 排查
├─ 學習新技術
└─ 安全審計

類別 3: 時間壓力（2 張）
├─ 緊急任務（< 1 小時）
└─ 充裕時間（> 1 天）

類別 4: 工具組合（3 張）
├─ 創業公司策略
├─ 企業級專案
└─ 個人開發者

類別 5: 錯誤決策識別（2 張）
├─ 過度使用 Claude Code
└─ 低估 Copilot 能力
```

## 15 張記憶卡內容

### 類別 1: 規模判斷

#### 卡片 1：小專案決策

```markdown
Q: 【情境】你要寫一個簡單的 CLI 工具，預計 500 行代碼，功能是解析 JSON 並輸出報表。
   完成時間：2 小時
   工具選擇？

A: 【決策】Copilot
   【理由】
   - 代碼規模小（< 1K 行）
   - 任務簡單明確
   - 時間緊迫，需要速度
   - 不需要深度架構設計

   【工作流程】
   1. 手動寫第一個函數（建立模式）
   2. Copilot 學習模式，快速補全其餘
   3. 總時間：< 1 小時完成

   【記憶點】
   小專案（< 1K 行）+ 簡單邏輯 = Copilot 夠用

   【錯誤選擇】
   - 用 Claude Code：等待時間累積，總時間可能變 2 小時
   - 用 Cursor：可以但不是最優（速度比 Copilot 慢）
```

#### 卡片 2：中專案決策

```markdown
Q: 【情境】開發一個內部管理系統，預計 20 個檔案，8,000 行代碼。
   包含：前端（React）+ 後端（Express）+ 資料庫（PostgreSQL）
   完成時間：2 週
   工具選擇？

A: 【決策】Cursor（主力）+ Copilot（輔助）
   【理由】
   - 規模適中（8K 行，20 檔案）
   - 需要跨檔案理解（前後端協同）
   - Cursor 的 32K context 足夠
   - 時間不緊迫，但也不能太慢

   【工作流程】
   Week 1: 用 Cursor 搭建架構和骨架
   Week 2: Cursor 處理跨檔案邏輯，Copilot 快速補全重複代碼

   【記憶點】
   中專案（1K-10K 行，< 30 檔案）= Cursor 最適合

   【何時切換】
   - 如果發現邏輯比預期複雜 → 考慮 Claude Code
   - 如果是簡單 CRUD → 多用 Copilot
```

#### 卡片 3：大專案決策

```markdown
Q: 【情境】重構一個電商平台，50,000 行代碼，80 個微服務。
   需要：遷移到新架構 + 優化性能 + 增加測試
   完成時間：3 個月
   工具選擇？

A: 【決策】Claude Code（必選，無替代）
   【理由】
   - 規模巨大（50K 行，80 檔案）
   - 只有 200K context 能理解全局
   - 需要 EPCV 工作流系統性重構
   - 風險高，不能有遺漏

   【工作流程】
   Month 1: Claude Code 深度分析 + 制定計劃
   Month 2-3: Claude Code 執行重構（可搭配 Cursor 做簡單任務）

   【記憶點】
   大專案（> 10K 行，> 50 檔案）= Claude Code 必選

   【成本考量】
   - API 費用：$500-1000
   - 但避免重構失敗的價值：> $50,000
   - ROI：> 50x
```

### 類別 2: 任務類型

#### 卡片 4：CRUD 開發

```markdown
Q: 【情境】你需要為 5 個資料模型各建立 CRUD API，共 20 個 endpoints。
   每個 endpoint 30-50 行代碼，模式高度重複。
   預計時間：3 小時
   工具選擇？

A: 【決策】Copilot
   【理由】
   - CRUD 是 Copilot 最擅長的場景
   - 高度重複，AI 學習快
   - 需要極速完成

   【效率對比】
   - Copilot：1-1.5 小時（70% 節省）
   - Cursor：2 小時
   - Claude Code：3-4 小時（反而更慢）

   【最佳實踐】
   1. 手動寫第一個 model 的完整 CRUD（建立模板）
   2. Copilot 學習模式
   3. 後續 4 個 model 幾乎全自動

   【記憶點】
   CRUD + 測試 + Boilerplate = Copilot 天下
```

#### 卡片 5：架構設計

```markdown
Q: 【情境】設計一個新的微服務架構，需要選擇技術棧、設計 API、規劃資料庫。
   團隊：5 人
   時間：2 週設計階段
   工具選擇？

A: 【決策】Claude Code（唯一選擇）
   【理由】
   - 需要深度思考 trade-offs
   - 需要對比多種方案
   - 需要生成完整設計文檔
   - Copilot 和 Cursor 都無法做技術決策

   【工作流程】
   Week 1: 用 Claude Code 深度分析、技術選型
   Week 2: Claude Code 生成設計文檔、API 規範

   【產出】
   - 技術選型報告（含 trade-offs 分析）
   - 架構設計圖
   - API 設計文檔
   - 資料庫 Schema
   - 風險評估

   【記憶點】
   架構設計 = Claude Code 專屬領域
   其他工具無法替代
```

#### 卡片 6：Bug 排查

```markdown
Q: 【情境】生產環境偶發性錯誤，涉及 4 個服務的交互，15,000 行代碼。
   錯誤日誌不明確，需要深度分析。
   時間：不緊急，但需要找到根本原因
   工具選擇？

A: 【決策】Claude Code
   【理由】
   - 跨服務問題，需要全局理解
   - 需要分析服務間交互
   - 可能是 Race Condition 等複雜 bug
   - 15K 行需要大 context

   【對比】
   簡單 Bug（單檔案，邏輯清晰）：
   └─ Copilot 或 Cursor 即可

   複雜 Bug（跨模組，原因不明）：
   └─ 必須 Claude Code

   【記憶點】
   Bug 難度 = 跨模組數量 × 代碼行數
   > 臨界值（10K 行或 5 模組）→ Claude Code
```

#### 卡片 7：學習新技術

```markdown
Q: 【情境】你需要學習 Rust 並開發一個 CLI 工具。
   你的 Rust 經驗：零
   時間：1 週學習 + 1 週開發
   工具策略？

A: 【決策】分階段策略
   【Week 1: 學習階段】
   └─ Claude Code 100%
      - 深度解釋概念（ownership, lifetime）
      - 互動式問答
      - 提供練習和檢查

   【Week 2: 開發階段】
   └─ Claude Code 30% + Copilot 70%
      - 遇到新概念問 Claude Code
      - 熟悉的模式用 Copilot 快速編碼

   【理由】
   - 學習需要深度理解（Claude Code）
   - 熟練後需要速度（Copilot）
   - 動態切換最優

   【記憶點】
   學習新技術 = 先深度（Claude）後速度（Copilot）
```

#### 卡片 8：安全審計

```markdown
Q: 【情境】即將上線的 Web 應用需要安全審計。
   檢查：SQL Injection, XSS, CSRF, 認證漏洞等
   代碼規模：20,000 行
   工具選擇？

A: 【決策】Claude Code（Security Auditor Agent）
   【理由】
   - 需要專業的安全知識
   - 需要系統性檢查所有漏洞
   - 需要生成詳細審計報告
   - Copilot 和 Cursor 無此能力

   【工作流程】
   1. 切換 Agent: claude /agents:security-auditor
   2. 深度掃描：檢查所有安全風險
   3. 生成報告：列出漏洞 + 修復方案

   【產出】
   - 安全漏洞清單（按嚴重性分級）
   - 修復代碼
   - 安全最佳實踐建議

   【記憶點】
   安全 + 效能 + 專業分析 = Claude Code Agent 模式
```

```markdown
A: 【決策】Copilot
   【理由】

   2. 定位錯誤函數
   3. Copilot 快速建議修復
   4. 測試 + 部署

   【記憶點】
   緊急 + 已定位 = Copilot（速度為王）
   緊急 + 未定位 = 先手動定位，再用 Copilot 修
```

```markdown
   Week 2:
   - Claude Code 執行重構

   【記憶點】
```

### 類別 5: 錯誤決策識別

#### 卡片 14：過度使用 Claude Code

```markdown
Q: 【反例】開發者小明所有任務都用 Claude Code，包括寫簡單的 getter 函數。
   問題在哪？應該怎麼改？

A: 【問題分析】
   - 簡單任務用 Claude Code = 殺雞用牛刀
   - 每次等待 20-30 秒，累積浪費大量時間
   - API 成本累積（簡單任務不值得）

   【實際影響】
   寫 10 個簡單函數：
   - 用 Claude Code：10 × 30 秒 = 5 分鐘（等待時間）
   - 用 Copilot：10 × 1 秒 = 10 秒
   差異：4 分 50 秒（每次）

   一天 100 次簡單操作：
   - 浪費時間：~50 分鐘
   - 浪費成本：$5-10

   【正確做法】
   - 簡單任務（CRUD, 測試）→ Copilot
   - 中等任務（功能開發）→ Cursor
   - 複雜任務（架構, 重構）→ Claude Code

   【記憶點】
   錯誤：All in Claude Code
   正確：根據任務複雜度選工具
```

#### 卡片 15：低估 Copilot 能力

```markdown
Q: 【反例】開發者小華認為 "Copilot 只是自動補全，不夠智能"，

   - 浪費時間在不需要深度思考的任務上

   【實際案例】
   任務：寫 50 個單元測試
   - 小華（手動）：5 小時
   - 用 Copilot：1.5 小時（節省 70%）

   Copilot 的真實能力：
   ✓ 學習你的代碼模式
   ✓ 生成高度一致的代碼
   ✓ 處理 CRUD, 測試, Boilerplate 極快
   ✓ 成本最低（$10/月無限使用）

   【正確認知】
   Copilot ≠ 簡單的補全工具
   Copilot = 重複性任務的效率引擎

   【使用場景】
   - CRUD API：Copilot 可能比 Claude Code 更快更好
   - 測試撰寫：Copilot 學習模式後幾乎全自動
   - Boilerplate：Copilot 的天下

   【記憶點】
   不要低估 Copilot，它在自己的領域是王者
```

## 使用指南

### Anki 設定

```
1. 下載 Anki（免費）
2. 建立新牌組："AI 工具選擇決策"
3. 將 15 張卡片匯入
4. 設定複習參數：
   - 新卡片/天：5 張
   - 複習卡片/天：20 張
   - 間隔：1d → 3d → 7d → 15d → 30d
```

### 學習節奏

```
Week 1：學習所有 15 張卡片（每天 3 張）
Week 2：複習 + 鞏固
Week 3-4：間隔重複（按 Anki 算法）
Month 2：偶爾複習（保持記憶）

目標：1 個月後建立快速決策直覺
```

### 自我測試

```
隨機抽取一張卡片的情境
→ 在 30 秒內做出決策
→ 說出理由
→ 檢查答案

達標標準：
- 30 秒內完成決策：✓
- 理由正確：✓
- 能說出何時切換工具：✓
```
