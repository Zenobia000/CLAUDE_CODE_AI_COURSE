# 課程改進

> 持續優化與迭代計劃

---

## 📖 目錄說明

此目錄用於收集**課程反饋、改進建議與迭代計劃**，確保課程內容持續優化並適應技術發展。

---

## 🎯 改進目標

### 核心原則

1. **數據驅動**: 基於學員反饋與學習成果數據進行改進
2. **持續迭代**: 小步快跑，定期更新課程內容
3. **適應變化**: 跟隨 AI 工具發展調整教學內容
4. **學員中心**: 以學員學習體驗為優先考量

## 📊 反饋收集機制

### 1️⃣ 學員反饋表

**每模組結束後收集**:

```markdown
# Module X 反饋表

## 課程內容（1-5 分）
- 理論深度: ⭐⭐⭐⭐⭐
- 實作難度: ⭐⭐⭐⭐⭐
- 內容實用性: ⭐⭐⭐⭐⭐
- 時間分配: ⭐⭐⭐⭐⭐

## 開放式問題
1. 最有幫助的部分是什麼？

3. 你希望增加哪些內容？
4. 你希望減少或移除哪些內容？

## 情境題庫評價
- 哪個情境最有挑戰性？
- 哪個情境最貼近實際工作？
- 是否有建議新增的情境？
```

### 2️⃣ 學習成果追蹤

**關鍵指標 (KPIs)**:

| 指標 | 目標值 | 測量方式 |
|------|--------|---------|
| **課程完成率** | ≥ 80% | 追蹤學員完成模組數 |
| **情境題完成率** | ≥ 70% | 統計各情境完成人數 |
| **專案品質** | ≥ 75 分 | 使用評分標準評估 |
| **滿意度** | ≥ 4.0/5 | 課後問卷調查 |
| **工具採用率** | ≥ 90% | 追蹤學員持續使用 AI 工具 |

## 🔄 改進流程

### 迭代週期

```
收集反饋 → 分析問題 → 設計改進方案 → 測試驗證 → 正式發布
    ↑                                                      ↓
    └──────────────────── 持續監控 ←─────────────────────┘
```

### 月度改進會議

**議程範例**:

```markdown
# 課程改進會議 - [年/月]

## 參與者
- 講師團隊
- 課程設計師
- （可選）學員代表

## 議程

### 1. 數據回顧（20 min）
- 本月學習數據分析
- 反饋表統計結果
- 完成率與滿意度

### 2. 問題識別（30 min）
- 重複出現的問題
- 學員建議整理
- 講師觀察分享

### 3. 改進提案（40 min）
- 針對問題提出解決方案
- 優先級排序
- 資源需求評估

### 4. 行動計劃（20 min）
- 確定改進項目
- 分配負責人
- 設定時間表

### 5. 下月計劃（10 min）
- 確認下次會議時間
- 待辦事項跟進
```

## 📝 改進建議分類

### 類型 A: 內容調整

**範例**:

```markdown
## 建議: Module 2 上下文管理內容過於複雜

**來源**: 5 位學員反饋，2 位講師觀察

**問題描述**:
- 學員對 .claudeignore 和 CLAUDE.md 的區別感到困惑
- 實作時間不足，理論講解過長

**改進方案**:
1. 將理論講解從 1 小時縮減為 40 分鐘
2. 增加對比表格，清楚說明兩者差異
3. 新增互動式練習，讓學員實際操作

**預期效果**:
- 理解度提升 20%
- 實作完成率提升至 90%

**負責人**: [姓名]
**預計完成**: [日期]
**狀態**: 計劃中 / 進行中 / 已完成
```

### 類型 B: 情境題庫優化

**範例**:

```markdown
## 建議: 新增 "多人協作 Git 衝突解決" 情境

**來源**: 學員建議，職場實用性調查

**理由**:
- 80% 學員在實際工作中會遇到 Git 衝突
- 現有課程缺少此情境
- 可結合 AI 工具加速衝突解決

**設計草案**:
**情境**: 團隊專案中，你的分支與主分支產生衝突...
**目標**: 使用 AI 工具分析衝突並安全合併
**時長**: 45 分鐘
**難度**: 組合級（Level 2）

**所需資源**:
- 準備範例專案（含預設衝突）
- 編寫解題步驟指引
- 錄製示範影片

**負責人**: [姓名]
**預計完成**: [日期]
```

### 類型 C: 工具更新

**範例**:

```markdown
## 建議: 更新 Claude Code 到最新版本

**觸發原因**:
- Claude Code v2.0 發布，新增 /agents 功能
- 現有教材基於 v1.5

**影響評估**:
- Module 2: 需新增 agents 使用說明（+ 30 min）
- Module 9: Security agent 可簡化安全掃描流程（- 15 min）
- 情境題庫: 3 個情境需調整

**更新計劃**:
1. Week 1: 更新理論文檔
2. Week 2: 調整情境題庫
3. Week 3: 更新範例代碼
4. Week 4: 講師培訓新功能

**負責人**: [姓名]
**預計完成**: [日期]
```

### 類型 D: 教學方法改進

**範例**:

```markdown
## 建議: 採用翻轉教室模式

**來源**: 教學法研究，講師實驗

**現狀問題**:
- 課堂時間有限，理論講解佔用過多時間
- 學員吸收程度不一，難以調整進度

**改進方案**:
- 課前發布預習影片（10-15 min）
- 課堂時間用於實作與答疑
- 設置課前小測驗，確保預習

**試點計劃**:
- 選擇 Module 3 進行試點
- 對比試點班與對照班的學習成果
- 收集學員與講師反饋

**成功指標**:
- 課堂實作時間增加 30%
- 學員滿意度提升 ≥ 0.5 分
- 知識保留率提升（課後 1 週測試）

**負責人**: [姓名]
**預計完成**: [日期]
```

## 📈 改進案例研究

### 案例 1: Module 2 上下文管理簡化

**背景**:
- 學員反饋理論過於複雜，實作時間不足
- 完成率僅 65%，低於目標 70%

**改進措施**:
1. 縮減理論講解時間（1h → 40min）
2. 新增對比表格與互動練習
3. 情境難度調整（3 個基礎級 → 2 個基礎級 + 1 個組合級）

**成果**:
- 完成率提升至 85%
- 滿意度從 3.8 提升至 4.3
- 學員反饋「更實用、更容易上手」

### 案例 2: 新增 Cursor IDE 教學

**背景**:
- 30% 學員在工作中使用 Cursor 而非 VS Code
- 原課程僅涵蓋 VS Code + Copilot

**改進措施**:
1. 新增 Module 3.5: Cursor IDE 專題（1.5 小時）
2. 創建 Cursor 專屬情境題庫（5 個）
3. 對比 Cursor vs VS Code 的使用場景

**成果**:
- 工具覆蓋率提升至 95%
- Cursor 用戶滿意度從 3.5 提升至 4.5
- 吸引更多 Cursor 用戶報名

### 案例 3: 翻轉教室試點

**背景**:
- 課堂時間緊張，理論講解佔用過多實作時間
- 學員希望更多動手練習

**改進措施**:
1. 錄製 Module 5 預習影片（15 分鐘）
2. 課前小測驗（5 題選擇題）
3. 課堂改為 90% 實作 + 10% 答疑

**成果**:
- 課堂實作時間增加 40%
- 知識保留率提升 25%（1 週後測試）
- 學員滿意度提升至 4.6（原 4.1）

## 📊 改進效果評估

### 評估方法

**定量指標**:
```markdown
改進前後對比:
- 完成率: 65% → 85% (↑ 20%)
- 滿意度: 3.8 → 4.3 (↑ 0.5)
- 測試分數: 72 → 81 (↑ 9 分)
```

**定性反饋**:
```markdown
學員訪談記錄:
"改進後的課程更貼近實際工作場景..."
"情境題庫難度更合理，循序漸進..."
```

### A/B 測試

```markdown
對照組 vs 實驗組:
- 對照組: 使用原課程內容
- 實驗組: 使用改進後內容
- 樣本數: 各 30 人
- 測試時長: 4 週
- 評估指標: 完成率、滿意度、知識保留率
```
