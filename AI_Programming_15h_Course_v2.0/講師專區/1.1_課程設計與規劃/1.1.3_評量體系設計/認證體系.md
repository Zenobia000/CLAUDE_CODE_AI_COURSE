## 🎯 認證哲學

### 為什麼需要認證？

**對學習者**：
```
✓ 驗證學習成果
✓ 獲得市場認可
✓ 建立學習動機
✓ 展示專業能力
```

**對雇主**：
```
✓ 快速識別合格人才
✓ 降低招聘成本
✓ 標準化能力評估
✓ 減少培訓投入
```

**對課程**：
```
✓ 建立品質標準
✓ 提升課程價值
✓ 促進持續改進
✓ 形成學習社群
```

### 認證設計原則

**1. 能力本位（Competency-Based）**
```
不是「學了多少」而是「會做什麼」

認證標準：
- 能獨立完成真實項目
- 能解決實際問題
- 能持續學習與進步
```

**2. 分級認證（Tiered Certification）**
```
不是「通過/失敗」而是「銅/銀/金」

讓每個人都能獲得相應等級的認可
鼓勵持續進步與挑戰
```

**3. 持續驗證（Continuous Validation）**
```
不是「一次認證終身有效」而是「持續學習與更新」

技術更新快，認證也需要更新
鼓勵終身學習
```

## 🔄 認證更新機制

### 為什麼需要更新？

```
AI 工具快速演進：
- Claude Code 新版本發布
- 新的 AI 工具出現
- 最佳實踐更新

認證需要保持時效性：
- Bronze: 1 年有效期
- Silver: 2 年有效期
- Gold: 3 年有效期
```

### 更新方式

**選項 A：持續學習積分（CPD - Continuing Professional Development）**

```yaml
積分要求:
  Bronze 更新: 10 CPD 積分/年
  Silver 更新: 20 CPD 積分/年
  Gold 更新: 30 CPD 積分/年

獲取積分方式:
  - 完成新情境：1 分/個
  - 參加工作坊：5 分/次
  - 技術文章：3 分/篇
  - 指導他人：2 分/人
  - 開源貢獻：5 分/個
  - 社群演講：10 分/次
```

**選項 B：重新認證**

```yaml
如果 CPD 積分不足：
  - 可選擇重新參加認證考試
  - 考試內容包含新增內容
  - 通過即可更新認證
```

**選項 C：升級認證**

```yaml
自動更新：
  - Bronze 升級到 Silver → Bronze 自動更新
  - Silver 升級到 Gold → Silver 自動更新

雙倍有效期：
  - 持有更高等級認證者
  - 低等級認證自動延長
```
